# v0.10.2 — PATCH release scope

Bundles findings from the external field review (2026-04-22, based on
`main@784eb01`), the internal phase C fresh-eyes code review
(2026-04-22), and the RELEASE_CHECKLIST staleness noticed during phase
E audit. No schema or behavioural changes — all items are bug fixes,
documentation gaps, or safety-net hardening.

## Context

v0.10.1 was a distribution-readiness patch (hardcoded path + i18n +
uninstall symlinks). Immediately after tagging, phase C review found
one more Russian-string leak (`memory-error-detect.sh`, landed as a
follow-up commit on `main`), seven hook-layer and Go-layer sharp edges
that don't block anyone but deserve to be tracked, and the external
reviewer's field observations identified a real `pinned`-eviction bug
and several nice-to-haves.

Rather than quietly living on `main` between tags, collect these into
a deliberate PATCH release so the external reviewer and future users
upgrade to a coherent snapshot.

## Scope

### High

- **Pinned files silently evicted from SessionStart injection by
  domain pressure.** External review: `user-profile` (status: pinned,
  knowledge, domains: [general]) was omitted from `fly-next` CWD
  injection while included from `circs` or `~`. Likely cause: the
  12-slot flex pool gives 9 active project domains stronger scores,
  pushing generic-domain pinned records out. CLAUDE.md spec says
  pinned files are exempt from rotation but doesn't state whether
  they're protected from injection-time eviction. Fix direction:
  reserve 1–2 pool slots for pinned records before domain-weighted
  scoring decides the rest.

- **Directory-driven `uninstall.sh` refactor.** v0.10.1 fixed the
  immediate symptom (three symlinks left behind on fresh-HOME
  installs) with a minimal list-append, but the underlying pattern —
  hand-rolled symlink list that falls out of sync with
  directory-driven `install.sh` — will keep biting. Refactor: walk
  `$HOOKS_DIR` / `$BIN_DIR`, remove any symlink whose target resolves
  into our repo, leave regular files alone.

### Medium

- **`hooks/memory-index.sh:55` TF-IDF tokenizer is Latin-only.**
  `gsub(/[^a-zA-Z\300-\377]/, " ", text)` truncates Cyrillic / CJK /
  Greek / Arabic. Records without `keywords:` whose bodies are in
  those scripts silently don't make it into the index. `internal/fts/
  tokenize.go` already uses `unicode.IsLetter` — i.e. the bash and
  Go paths diverge on tokenization rules, and parity-check doesn't
  cover memory-index. Options: (a) port TF-IDF to Go and unify, (b)
  update the bash regex to use a POSIX class or a sed-equivalent, (c)
  accept the limitation and document it. Proposal: (a), small port,
  closes the bash/Go divergence.

- **`internal/dedup/dedup.go:119` WAL-before-disk-op ordering.**
  `wal.Append(dedup-merged)` + Fprintf run regardless of whether
  `incrementRecurrence(bestFile)` succeeded. If the increment fails
  (permissions, I/O), WAL says "merged" but neither the recurrence
  bump nor the file deletion happened. Move both into the `if err ==
  nil` branch.

- **`hooks/memory-analytics.sh` undocumented formulas and thresholds.**
  `eff = (pc + 1) / (pc + nc + 2)` is Laplace smoothing (pseudo-counts
  for Bayesian mean); `0.7` / `0.3` / `3` / `5` are the winner / noise
  / min-injects thresholds. No comment explaining why. Expose
  thresholds via `.config.sh` (new vars: `ANALYTICS_WINNER_THRESHOLD`
  etc.) and add a comment block explaining the formula.

- **`RELEASE_CHECKLIST.md` Docker bootstrap references removed
  `uv`.** v0.10.0 dropped uv; the Docker test step still says
  `curl -LsSf https://astral.sh/uv/install.sh | sh`. Update the test
  to install `go` instead (for `make build`) and drop the uv line.

### Low (nice-to-haves from external review Part 4)

- **Execute bits on `hooks/test-memory-hooks.sh` and `hooks/bench-
  memory.sh`.** Currently non-executable despite shebangs. `chmod +x`
  at install time or commit with the bit set.

- **`install.sh` end-of-run summary.** `./install.sh --skip-base`
  exits silently. One-line summary ("Linked N hooks, M bin scripts,
  patched K CLAUDE.md files") would confirm the install.

- **`.gitignore` covers runtime artefacts.** Currently `.DS_Store`,
  `*.bak`, `*.tmp`. Adds: `.analytics-report`, `.wal*`, `index.db`,
  `.runtime/`, `self-profile.md`. Only matters if someone turns
  `~/.claude/memory/` into a personal backup repo — preventative.

- **`hooks/memory-outcome.sh:60-72` duplicates parse-memory.sh
  logic.** Inline awk parses `related:` / `instance_of:` instead of
  using `hooks/lib/parse-memory.sh`. Refactor to library call.

- **`hooks/session-stop.sh` decomposition.** 700 lines mixing
  evidence match, outcome detection, rotation, and reminders.
  Partial decomposition exists (`evidence-extract.sh`). Split into
  `outcome-detection.sh`, `rotation.sh`, `reminder-builder.sh`. Pure
  refactor — no behavioural change.

- **`docs/TROUBLESHOOTING.md` + `docs/FAQ.md` ambient gap.**
  External review: `precision_class: ambient` lives only in README.
  A user reading self-profile and wondering why certain rules only
  appear under "ambient activations" has no index entry outside the
  full README. Add a 3-line entry in TROUBLESHOOTING.

## Non-goals

- No schema changes to frontmatter or WAL.
- No new memory types.
- No changes to ranking formulas themselves (only their exposure to
  `.config.sh`).
- Not a MINOR release — v0.11 decisions-autoreflection and v0.12
  evidence-learn remain separate plans with their own scope.

## Implementation steps

Ordered by dependency, not priority:

1. **Fix `dedup.go` WAL ordering** — one `if` block move, test that
   the failed-increment case now skips the WAL. Safe.
2. **Port TF-IDF `memory-index.sh` to Go as `memoryctl tfidf
   rebuild`.** Extend parity-check to cover it. Bash fallback stays
   with current Latin-only behaviour flagged as "known limitation".
3. **Pinned injection slot reservation.** Requires understanding the
   flex-pool assembly path in `build-context.sh`. Design before code.
   Test case: dry-run from `fly-next` CWD must include `user-profile`
   post-fix. External reviewer's simulation is the exact reproducer.
4. **Directory-driven uninstall.** Read `$HOOKS_DIR` / `$BIN_DIR`,
   resolve each symlink target, drop if inside `$SCRIPT_DIR`. Add
   integration test: fresh-HOME install + uninstall + assert 0
   residual symlinks.
5. **`.config.sh` analytics thresholds.** New vars + comment block in
   `hooks/lib/load-config.sh` parse list + fallback defaults in
   `memory-analytics.sh`.
6. **RELEASE_CHECKLIST Docker update.** One-line diff.
7. **`.gitignore`, execute bits, install summary, outcome refactor,
   session-stop split.** Five small commits. Session-stop split goes
   last because it touches the largest file — easier to review when
   other changes are already merged.
8. **TROUBLESHOOTING + FAQ ambient entry.** One commit.

## Open questions

- **Pinned-slot policy details.** Should pinned files claim a
  reserved slot in the per-type flex pool (e.g., guaranteed 1 of 4
  knowledge slots), or in the overall budget (MAX_FILES)? Proposal:
  per-type. Overall budget already has project/global mistake
  reservations — similar pattern.
- **TF-IDF bash fallback fate.** If we port to Go, does the bash
  version stay as a fallback (like self-profile has both) or get
  removed with a dep on `memoryctl tfidf`? Proposal: keep bash as
  fallback with Latin-only behaviour documented as known-limitation —
  no user loses functionality, Russian/CJK users benefit from the Go
  binary.
- **Analytics threshold names in `.config.sh`.** `WINNER_EFF` /
  `NOISE_EFF` / `MIN_INJECTS_WINNER` / `MIN_INJECTS_NOISE`? Or
  prefixed `ANALYTICS_*`? Proposal: `ANALYTICS_*` prefix to avoid
  collision with future ranking vars.
