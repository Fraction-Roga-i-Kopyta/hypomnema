# v0.9 — Decomposition of session-start.sh

> Deferred from v0.8 (Task 17 in `2026-04-15-v0.8-cleanup-and-onboarding.md`). The decomposition is risky and deserves a fresh session with full attention.

**Goal:** Shrink `hooks/session-start.sh` from 1286 lines to ≤300 by extracting three focused modules under `hooks/lib/`. All 200 existing smoke tests must remain green.

## Why deferred

- ~50 global variables flow between blocks (`KEYWORDS`, `DOMAINS`, `PROJECT`, `WAL_SCORES`, `TFIDF_SCORES`, `INJECTED_FILES`, `INJECTED_SLUGS`, `_PRIORITY_SLUGS`, `CONTRADICTS_WARNINGS`, `CLUSTER_*`, `MISTAKES_MD`/`FEEDBACK_MD`/etc., `_EARLY_EMPTY_CONTEXT`, `_TOTAL_CAP`, …).
- AWK programs (`AWK_MISTAKES`, `AWK_SCORED`) live as multi-line HEREDOCs inside the bash flow — moving them changes how the surrounding state is consumed.
- `collect_scored()` is called five times with different `cap` parameters but mutates a shared `_SCORED_USED` accumulator and `MAX_SCORED_TOTAL`.
- Each extraction risks breaking one of 200 tests; mitigation is a per-extraction full-suite run.

## Target structure

```
hooks/lib/
├── parse-memory.sh    # frontmatter parsing, file enumeration, AWK programs
├── score-records.sh   # keyword + WAL + TF-IDF + project-boost scoring
├── build-context.sh   # markdown assembly, section markers, cluster expansion
└── (existing) wal-lock.sh, stat-helpers.sh, detect-project.sh, evidence-extract.sh
```

`session-start.sh` becomes orchestration only (~280 lines):
1. Parse input, detect project, detect domains.
2. Source lib modules.
3. Call `collect_*` from `parse-memory.sh`.
4. Call `score_*` from `score-records.sh`.
5. Call `render_section` / `mark_contradicts` / `expand_clusters` from `build-context.sh`.
6. Emit final JSON.

## Approach

Three sub-tasks, each its own commit, each followed by full suite run.

### 17a. Extract parsing — `lib/parse-memory.sh`

Move:
- `parse_related()` (line 276)
- `find_memory_file()` (line 295)
- `domain_matches()` (line 307)
- `AWK_MISTAKES` and `AWK_SCORED` HEREDOCs (lines 200-275)
- `enumerate_*` style helpers (currently inline in `collect_*` calls)

Risks:
- AWK HEREDOCs reference shell variables via `-v` flags — pass them explicitly through function args, not globals.
- BSD awk `FNR==1` boundary detection still required (no BEGINFILE/ENDFILE).

### 17b. Extract scoring — `lib/score-records.sh`

Move:
- WAL parsing loop (~lines 460-548)
- TF-IDF index loading (~lines 519-540)
- `score_record()` formula
- Noise penalty calculation
- `collect_scored()` and its call sites

Risks:
- Shared `_SCORED_USED` accumulator must become an explicit nameref or returned value.
- `MAX_SCORED_TOTAL` budget tracking — replace global mutation with explicit counter passed by reference.

### 17c. Extract context assembly — `lib/build-context.sh`

Move:
- Section rendering loops (`MISTAKES_MD`, `FEEDBACK_MD`, etc.)
- `[ПРИОРИТЕТ]` perl pass (Task 1 escape already in place)
- Cluster expansion (forward + reverse + cross-injected scans, lines 736-976)
- `[REVIEW: ...]` cascade markers
- Final `CONTEXT` assembly

Risks:
- Many section vars must become structured data (associative array of section name → markdown).
- Cluster code is the most tangled — depth limit and capping during extraction would also help.

## Test discipline

After each sub-task:
1. `bash hooks/test-memory-hooks.sh` — must be 200/200.
2. `bash hooks/bench-memory.sh` — must stay within 20% of baseline.
3. Manual smoke: `echo '{"session_id":"x","cwd":"/tmp"}' | bash hooks/session-start.sh | jq -r '.hookSpecificOutput.additionalContext'` — must be non-empty for a populated memory.
4. Commit. Do not bundle multiple sub-tasks in one commit.

## Bonus during decomposition

- Add depth limit to `related:` cluster expansion (max 1 hop) — currently can chain unboundedly.
- Cap `contradicts` markers at 2 per session (visual noise).
- Generate a `[ПРИОРИТЕТ → newer is X]` recommendation instead of raw warning.
- Externalize remaining decay-rate constants from `session-stop.sh` into `.config.sh`.

## Estimated effort

4-6 hours of focused work in a fresh session with the codebase fully loaded. Do not attempt at the tail of a long working session — too many globals to track without losing one.
