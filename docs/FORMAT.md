# Hypomnema storage format ŌĆö contract v1.0

**Status:** STABLE. Any change to this document is a breaking change to the memory engine and requires a major version bump.

This document describes what's on disk. It is the shared surface between the reference bash implementation and any alternate implementations (a Go port, Rust bindings, custom consolidator, etc.). Anything outside this document ŌĆö AWK program text, specific hook scripts, ranking formula constants ŌĆö is implementation detail and may change between minor versions.

If you are writing code that reads or writes `~/.claude/memory/`, conform to this document. If you need behaviour this document does not describe, propose an extension; do not invent silently.

---

## 1. Version

Every implementation declares a format version it understands:

```
format_version: 1
```

This document describes **format 1**. A file or WAL produced under format 1 is valid input for any format-1 reader. Additive changes (new optional fields, new WAL event types) stay at format 1. Removing, renaming, or changing the semantics of an existing field increments the major version.

Implementations MUST:
- Accept format-1 input and produce format-1 output.
- Ignore unknown frontmatter fields (forward-compatibility).
- Log, not crash, on unknown WAL event types (see ┬¦ 5).

## 3. Universal frontmatter

Every memory file is a Markdown document beginning with a YAML frontmatter block delimited by `---` lines. All types share these fields:

| Field | Required | Type | Notes |
|---|:-:|---|---|
| `type` | yes | enum | One of `mistake \| strategy \| feedback \| knowledge \| decision \| note \| project \| continuity`. |
| `project` | yes | string | Either `global` or a project slug matching an entry in `projects.json`. |
| `created` | yes | YYYY-MM-DD | Immutable. Used for recency ranking in UserPromptSubmit. |
| `status` | yes | enum | One of `active \| pinned \| stale \| archived`. `pinned` protects from rotation/archival ŌĆö see ┬¦ 3.1 for what it does NOT guarantee. |
| `description` | no | string | One-line summary. Shown in MEMORY.md index. |
| `keywords` | no | list | Tokens used in keyword-match scoring. Inline `[a, b]` or block `- a\n- b`. |
| `domains` | no | list | Same form as `keywords`. Filters files by project domain. |
| `injected` | auto | YYYY-MM-DD | First-injected timestamp. Writers SHOULD NOT set it by hand. |
| `referenced` | auto | YYYY-MM-DD | Refreshed on every SessionStart injection. |
| `ref_count` | auto | integer | Incremented on every injection. Start at 0. |

**Field ordering** is not semantically significant. Implementations SHOULD preserve order on round-trip but MUST accept any order on read.

### 3.1 `status: pinned` ŌĆö what it guarantees

`pinned` is narrow by design. It guarantees:

- The file is **never rotated** (`active ŌåÆ stale ŌåÆ archived`) by the lifecycle hook, regardless of age or `decay_rate`.
- `mistake` files with `recurrence: 0` still inject when pinned (default would skip them).
- A file with malformed frontmatter is preserved (the fallback archival path skips pinned files explicitly).

It does NOT guarantee:

- A reserved slot in SessionStart's injection output. A pinned knowledge file competing against many higher-scored candidates can still lose its slot to keyword-match winners or WAL-score winners. The flex-pool (12 / 10 / 8 slots depending on keyword signal strength) is allocated by score, not by status.
- A reserved slot in UserPromptSubmit's trigger-matched output. The priority key is `project_rank ŌåÆ status ŌåÆ severity ŌåÆ recency ŌåÆ logŌéüŌéĆ(ref_count) ŌåÆ recurrence ŌåÆ ref_count`; `pinned` outranks `active` but a pinned file with zero keyword match and a low priority key can still be ordered below a project-matched triggered file.
- Immunity to WAL aggregation. `inject` events on pinned files are compacted into `inject-agg` like any other.

If you need a file that is guaranteed to surface on every SessionStart regardless of the current domain / keyword mix, the current contract does not express that. The workaround is to set `scope: universal` (for mistakes) and add broad `triggers:` that the user's typical prompts will match ŌĆö but neither is a hard guarantee.

**`auto` fields** are managed by hooks. Users and external writers MAY leave them unset; implementations MUST preserve them if present and update them on inject events. New templates SHOULD ship with `ref_count: 0` so the auto-increment pass finds the key.

**Tolerated syntax variants** (parser MUST accept all):

- Tab after a key colon: `status:\tactive` parses the same as `status: active`.
- Quoted scalars: `status: "active"` and `status: 'active'` parse as `active`.
- Block-style lists (`keywords:\n  - a\n  - b`) flatten to the same space-separated string as inline `[a, b]`.
- UTF-8 BOM at file start is stripped silently.
- CRLF line endings are normalized to LF.

**Not supported** (parser SHOULD reject or substitute a sentinel):

- Block-scalar markers (`|` or `>`) on scalar-valued fields. Parser substitutes `_multiline_unsupported_`.
- Unclosed frontmatter (missing second `---`). File is skipped entirely from the main scoring pass; logged as `schema-error`.

## 4. Per-type extensions

### `type: mistake`

| Field | Required | Type | Notes |
|---|:-:|---|---|
| `severity` | yes | enum | `critical \| major \| minor`. Unset treated as minor-adjacent for ranking. |
| `recurrence` | yes | integer | Count of observed repetitions. Pinned files with `recurrence: 0` still inject. |
| `scope` | no | enum | `universal \| domain \| narrow` (default `domain`). `narrow` files inject only on keyword match. |
| `root-cause` | yes | single-line | What went wrong. One line; use body for prose. |
| `prevention` | yes | single-line | How to avoid next time. One line. |

### `type: strategy`

| Field | Required | Type | Notes |
|---|:-:|---|---|
| `trigger` | either | single-line | Substring cue. Matched case-insensitively against user prompt. |
| `triggers` | either | list | Multiple cues. Either-or with `trigger`; accept both; merge into one phrase list. |
| `success_count` | no | integer | Starts at 0. Bonus at `min(success_count ├Ś 2, 6)` in SessionStart score. |

Both `trigger:` (singular, legacy) and `triggers:` (array, current) MUST be accepted. Phrases are substring-matched against the cleaned prompt (fences, inline code, and blockquotes stripped).

### `type: feedback`

| Field | Required | Type | Notes |
|---|:-:|---|---|
| `evidence` | no | list | Phrases that signal the rule was applied. Case-insensitive substring match. |
| `precision_class` | no | enum | `ambient` excludes the file from the measurable-precision denominator. |

### `type: knowledge` / `decision` / `note`

| Field | Required | Type | Notes |
|---|:-:|---|---|
| `related` | no | list | Other slugs this file clusters with. Inline `[slug-a, slug-b:contradicts]` or block form. `:type` suffix annotates the relationship. |

### `type: project`

No additional required fields. Body is a human-readable project overview.

### `type: continuity`

Auto-generated by the Stop hook. Body contains a "Git State" section (branch, last commit, dirty status) and a "Next step" block. Writers outside the hook SHOULD NOT create these by hand.

## 5. WAL grammar

The WAL (`~/.claude/memory/.wal`) is an append-only plaintext log. Each line has exactly **four** pipe-delimited fields:

```
date|event|target|session
```

| Field | Shape |
|---|---|
| `date` | `YYYY-MM-DD`, local timezone of the writing machine. |
| `event` | Lowercase kebab-case identifier. See table below. |
| `target` | Slug or identifier relevant to the event. Pipes and newlines in source data MUST be replaced with `_`. |
| `session` | Session identifier, pipe/slash-sanitised to `_`. |

**Field count is invariant**: tools (readers, compactors) rely on `awk -F'|'` with exactly four fields. If a future event needs richer data, encode it into `target` with a sub-delimiter (`,` or `>`) ŌĆö do NOT add a fifth column.

### 5.1 Defined event types

**Core injection lifecycle:**

- `inject` ŌĆö SessionStart injected a file. `target` = slug.
- `inject-agg` ŌĆö WAL compaction folded multiple `inject` events into one. `target` = `slug,count` (comma-separated).
- `referenced` ŌĆö metadata bump without content change. Rare; used by out-of-band touches.

**Retrieval signals:**

- `trigger-match` ŌĆö UserPromptSubmit matched a substring trigger. `target` = slug.
- `trigger-useful` ŌĆö Stop hook detected a citation of an injected file. `target` = slug.
- `trigger-silent` ŌĆö Stop hook completed without a citation. `target` = slug.
- `shadow-miss` ŌĆö FTS5 shadow pass surfaced a file the primary pipeline missed. `target` = slug. Diagnostic-only; MUST NOT affect injection.

**Outcomes:**

- `outcome-new` ŌĆö a new mistake file was just written this session. `target` = slug.
- `outcome-positive` ŌĆö injected mistake was not repeated in this session. `target` = slug.
- `outcome-negative` ŌĆö injected mistake was repeated. `target` = slug.
- `clean-session` ŌĆö session ended with no errors. `target` = domain or `_global_`.

**Session rollup:**

- `session-metrics` ŌĆö `target` = comma-separated `key:value` pairs (`error_count:N,tool_calls:M,duration:Ss`). A reader MUST tolerate unknown keys.

**Strategy signals:**

- `strategy-used` ŌĆö a strategy file was cited by Claude. `target` = slug.
- `strategy-gap` ŌĆö a clean session where no strategy was recorded. `target` = domain.
- `strategy-rescored` ŌĆö `bin/memory-strategy-score.sh` rebuilt `success_count`. `target` = slug.

**Dedup:**

- `dedup-blocked` ŌĆö PreToolUse blocked a write. `target` = `new-slug>existing-slug`.
- `dedup-merged` ŌĆö post-hoc merge. `target` = `new-slug>existing-slug`.
- `dedup-candidate` ŌĆö mid-similarity warning. `target` = `new-slug~existing-slug`.

**Cluster / cascade:**

- `cluster-load` ŌĆö a related file was injected via cluster expansion. `target` = `src-slug>target-slug`.
- `cascade-review` ŌĆö cascade outcome flagged for user review. `target` = `parent-slug:child-slug`.

**Lifecycle:**

- `rotation-stale` ŌĆö file marked stale by age threshold. `target` = slug.
- `rotation-archive` ŌĆö file moved to archive/. `target` = slug.
- `rotation-summary` ŌĆö summary of a rotation pass. `target` = `stale:N,archive:M`.

**Schema / health:**

- `schema-error` ŌĆö a memory file failed to parse. `target` = slug.
- `format-unsupported` ŌĆö a memory file declared `format_version: N` with `N > 1`; the file is skipped and not surfaced to Claude. `target` = `slug:N`. De-duplicated per slug per day by the emitting hook.
- `evidence-missing` ŌĆö a feedback file's listed `evidence:` file is absent. `target` = phrase-hash.
- `evidence-empty` ŌĆö a feedback file's body yielded zero content tokens. `target` = slug.
- `error-detect` ŌĆö PostToolUse detected a stderr/stdout error pattern. `target` = `tool:category`.
- `index-stale` ŌĆö a derivative index (TF-IDF, FTS) was found stale. `target` = index name.

### 5.2 Extension rules

- Adding a new event type is non-breaking. Readers MUST ignore (not crash on) unknown events.
- The four-column invariant is immutable within format v1. A fifth column requires a major version.
- New events SHOULD follow the naming pattern `<domain>-<action>` (e.g., `cluster-load`, not `load_cluster`).

## 6. Derivative indices

These files are rebuildable from markdown + WAL:

- `.tfidf-index` ŌĆö TSV: `term\tfile:score\tfile:score\tŌĆ”`. Locale-stable decimals (`.`), one line per term.
- `index.db` ŌĆö SQLite with an FTS5 virtual table over body content. Schema is an implementation detail; drift check is `mtime(index.db) >= mtime(newest .md)`.
- `.analytics-report` ŌĆö Markdown report emitted by `bin/memory-self-profile.sh` (or its Go equivalent).

Any implementation MAY delete these files at any time and rebuild them on next access. Conforming implementations MUST tolerate their absence.

## 7. Parsing robustness

### Accept

- Tab after key colons (`status:\tactive`).
- Quoted scalars (`status: "active"`, single-quoted same).
- Inline arrays (`keywords: [a, b, c]`) and block arrays (`keywords:\n  - a\n  - b`).
- UTF-8 BOM at file start (stripped).
- CRLF line endings (normalized to LF on read).
- Missing optional fields (fall back to documented defaults).

### Reject or sentinel

- Unclosed frontmatter (missing second `---`) ŌåÆ skip file from scoring; log `schema-error`.
- Block scalar markers (`|` or `>`) on scalar fields ŌåÆ substitute `_multiline_unsupported_`.
- Pinned files with malformed frontmatter ŌåÆ do NOT archive (bypasses the mtime fallback).

### Case sensitivity

- Field keys are lowercase ASCII. `STATUS: active` does NOT parse. This is intentional ŌĆö it's a schema contract, not YAML per se.
- Values are preserved as-is except where documented (quoted stripping, list flattening).

## 8. Backward compatibility

An implementation conforming to format v1 MUST:

1. **Accept** all fields and event types defined in this document.
2. **Preserve** unknown frontmatter fields on round-trip (write back what you read).
3. **Tolerate** unknown WAL event types (log, do not crash).
4. **Never silently drop** a `status: pinned` file (that file is user-protected).

An implementation MAY:

- Add new optional frontmatter fields (writes become partially-forward-incompatible ŌĆö other readers will see unknown fields and preserve them).
- Emit new WAL event types (readers will tolerate; consumers of the specific event will need updating).
- Regenerate derivative files at any time.

An implementation MUST NOT:

- Rename an existing field or event type.
- Change the meaning of an existing field or event type.
- Break the WAL four-field invariant.
- Delete archive/ content.

Breaking any of the MUST NOT rules requires incrementing `format_version`.

---

## 9. Extension process

If you need a new frontmatter field or WAL event type, the steps are:

1. Open a PR adding the field/event to this document **first**.
2. List: what it means, who writes it, who reads it, default behaviour when missing.
3. Update the reference bash implementation to emit/read the new entry.
4. Tag a minor version release.

Do not add a field to memory templates and hope readers pick it up. Do not emit a new WAL event type from a one-off script and expect rotation to handle it.
