# v0.7.1 — Feedback detector redesign

**Status:** design approved, pending implementation plan
**Date:** 2026-04-15
**Scope:** patch release, hooks/session-stop.sh + new lib

## Problem

Feedback loop introduced in v0.6 writes `trigger-useful` / `trigger-silent` to WAL after each session, intended to feed downstream auto-disable of noisy triggers (per-trigger precision).

Six months of dogfooding produced:

- **1472 injects**
- **0 `trigger-useful`**
- **39 `trigger-silent`**

Signal is fully dead. Cause is in `hooks/session-stop.sh:486`:

```sh
if printf '%s' "$_ASSIST_TEXT" | grep -qF "$_inj_slug" 2>/dev/null; then
  # trigger-useful
```

Detector does literal `grep -F` for the slug (filename without `.md`) in assistant text. In practice the assistant applies the rule silently — `"add a tz-aware datetime"` — without ever naming `code-approach`. Every injected memory gets marked silent regardless of actual usefulness.

Any downstream auto-disable built on this signal will penalise working memories indiscriminately.

## Goals

1. Restore signal: after fix, at least one `trigger-useful` event per typical session where a memory applied.
2. Bias toward `useful` (low threshold) — false silents are costlier than false usefuls for downstream auto-disable.
3. Keep patch-scoped: no WAL schema change, no memory migration required.
4. Maintain 160/160 test baseline; replace broken Test 20 with a real one.

## Non-goals

- Graded scoring (0.0–1.0 instead of binary). Deferred to future release if binary proves insufficient.
- LLM-based semantic scoring. Adds cost and non-determinism.
- Explicit `<!-- uses: slug -->` markup from assistant. Relies on LLM discipline.
- Migration of all existing memories to new schema.

## Design

### Architecture

Replace literal slug grep with **content-based matching against memory body**. Two branches, tried in order:

1. **`evidence:` frontmatter branch** — if memory has an explicit `evidence:` YAML array, match its entries against assistant text. Threshold: ≥1 phrase matched.
2. **Body-mining branch (fallback)** — if no `evidence:` field, extract content tokens from memory body, match against assistant text. Threshold: ≥2 unique tokens matched.

Binary output preserved: `trigger-useful` or `trigger-silent`. WAL schema unchanged.

### Components

**New: `hooks/lib/evidence-extract.sh`** — sourced from `session-stop.sh`. Two functions:

- `evidence_from_frontmatter <path>` → stdout: newline-separated phrases from `evidence:` YAML array. Empty if field absent or malformed.
- `evidence_from_body <path>` → stdout: newline-separated unique tokens from body, lowercased, length ≥4, not in stop-list. Body = content after frontmatter closing `---`, excluding:
  - Lines starting with `**Why:**` or `**How to apply:**`
  - Fenced code blocks (between ` ``` `)
  - Blockquote lines (`>`)
  - The slug name itself (prevents self-reference inflation)

Both functions cap input at 10 KB (`head -c 10240`) to defend against pathologically long notes.

**Stop-list (built into evidence-extract.sh):**

```
ru: это что как для при если или можно нужно быть очень только также
    этот эта эти они она оно свои свой такой такая такие была были есть
    чего чтобы потому поэтому вообще просто всегда когда даже тоже уже
en: the this that with when from have will been into than then also
    what where which while would could should about their there these
    those such more some only many most just even only also much very
markdown: why how note example apply reason
```

Length ~50 entries. No external dependencies.

**Modified: `hooks/session-stop.sh:462-501`** — the loop over `_INJECTED_ALL`:

```sh
# (pseudo-code; exact impl in plan)
. "$(dirname "$0")/lib/evidence-extract.sh" 2>/dev/null || true

# Normalize assistant text once
_ASSIST_TEXT_LC=$(printf '%s' "$_ASSIST_TEXT" | tr '[:upper:]' '[:lower:]')

while IFS= read -r _inj_slug; do
  [ -z "$_inj_slug" ] && continue

  _MEM_FILE=$(find "$MEMORY_DIR" -maxdepth 3 -name "${_inj_slug}.md" 2>/dev/null | head -1)
  if [ -z "$_MEM_FILE" ]; then
    wal_append_event "evidence-missing|${_inj_slug}|${SAFE_SESSION_ID}"
    _FEEDBACK_LINES="${_FEEDBACK_LINES}${TODAY}|trigger-silent|${_inj_slug}|${SAFE_SESSION_ID}\n"
    continue
  fi

  _EVID=$(evidence_from_frontmatter "$_MEM_FILE")
  if [ -n "$_EVID" ]; then
    _HITS=$(count_substring_hits "$_ASSIST_TEXT_LC" "$_EVID")
    _THRESHOLD=1
  else
    _EVID=$(evidence_from_body "$_MEM_FILE")
    if [ -z "$_EVID" ]; then
      wal_append_event "evidence-empty|${_inj_slug}|${SAFE_SESSION_ID}"
      _FEEDBACK_LINES="${_FEEDBACK_LINES}${TODAY}|trigger-silent|${_inj_slug}|${SAFE_SESSION_ID}\n"
      continue
    fi
    _HITS=$(count_word_hits "$_ASSIST_TEXT_LC" "$_EVID")
    _THRESHOLD=2
  fi

  if [ "$_HITS" -ge "$_THRESHOLD" ]; then
    _FEEDBACK_LINES="${_FEEDBACK_LINES}${TODAY}|trigger-useful|${_inj_slug}|${SAFE_SESSION_ID}\n"
  else
    _FEEDBACK_LINES="${_FEEDBACK_LINES}${TODAY}|trigger-silent|${_inj_slug}|${SAFE_SESSION_ID}\n"
  fi
done <<< "$_INJECTED_ALL"
```

### Data flow

```
session-stop.sh reads WAL → _INJECTED_ALL (list of slugs in session)
           ↓
  for each slug:
           ↓
  resolve file path via find (maxdepth 3)
           ├── not found → silent + evidence-missing event
           ↓
  evidence_from_frontmatter(file)
           ├── non-empty → case-insensitive substring match in _ASSIST_TEXT
           │              hits ≥ 1 → useful, else silent
           │
           └── empty → evidence_from_body(file)
                      ├── empty after filtering → silent + evidence-empty event
                      └── non-empty → case-insensitive word-boundary match
                                       hits ≥ 2 → useful, else silent
           ↓
  append to _FEEDBACK_LINES → wal_run_locked → append to WAL
```

Invariants:

- `_ASSIST_TEXT` extracted once per session (existing behavior).
- Lowercasing done once on assistant text, once per evidence list.
- File path resolved via `find` — portable across categories (`mistakes/`, `strategies/`, `notes/`, `feedback/`).
- Empty `_ASSIST_TEXT` → all silent (existing behavior preserved).

### Error handling

Detector is best-effort; must not break `session-stop.sh` (critical for all telemetry).

| Case | Behavior |
|------|----------|
| Memory file not found | `trigger-silent` + `evidence-missing` WAL event |
| Both `evidence:` and body-mining empty | `trigger-silent` + `evidence-empty` WAL event |
| `evidence-extract.sh` source fails | `_FEEDBACK_LINES` not written; no crash (`\|\| true` on source) |
| awk/grep errors inside functions | `2>/dev/null \|\| true`; empty stdout; other slugs continue |
| Memory body > 10 KB | Truncated via `head -c 10240` |

New WAL event types introduced: `evidence-missing`, `evidence-empty`. De-duplicated per day per slug (same pattern as existing `schema-error`).

## Open questions

None. All design decisions confirmed during brainstorming session 2026-04-15.

## Risks

1. **Body mining too noisy for multi-rule memories** — `code-approach.md` contains 5 unrelated rules. Any session mentioning ANY of them → useful. Mitigation: if auto-disable later shows this memory as always-useful but irrelevant, that's a signal to split it into atomic rules (good practice anyway).
2. **Russian tokenization** — `tr '[:upper:]' '[:lower:]'` on UTF-8 Cyrillic relies on locale. Test on macOS default locale (en_US.UTF-8) and verify lowercase of `ПАРАМЕТРИЗОВАННЫЙ` → `параметризованный`. If fails, add `iconv`-based normalization.
3. **Token overlap for short assistant responses** — "ok", "done" responses give no tokens. These sessions would always silent. Acceptable: a session with a 5-char response wasn't meaningfully doing work.

## Downstream unblocked

With a working signal, v0.8 can implement:

- Per-trigger precision aggregation (`useful / (useful + silent)` windowed).
- Auto-demotion of triggers below precision threshold (e.g., <20% useful over 20+ injections).
- `self-profile.md` gains a "noisy triggers" section.
