# v2 Ranker A/B — does ranking beat random? (null-hypothesis replay)

**Date:** 2026-05-29
**Tool:** `memoryctl ab` (Phase 2b; `internal/ab`)
**Corpus:** maintainer's live WAL (`~/.claude/memory/.wal`) — 49 eval sessions

## Question

Does the v2 relevance ranker concentrate a session's useful memories into a small
injection budget better than random selection? (The "does ranking earn its keep"
question raised in the v2 repositioning analysis — never previously tested against
a null baseline.)

## Method

Temporal-holdout replay, leakage-free:
- **Ground truth** per session S: slugs that received `trigger-useful|slug|S`.
- **Signals** aggregated from WAL events **strictly before** S's date (ref_count incl.
  inject-agg counts, Bayesian effectiveness `(pos+1)/(pos+neg+2)`, recency vs S's date).
  No lookahead — a slug's score for S cannot be inflated by S itself.
- **Candidate pool** = all distinct slugs in the WAL.
- **Ranked** = `rank.Rank` top-K (static signals only; **Overlap=0** — historical prompt
  keywords were never stored, so the keyword-overlap component is NOT exercised here).
- **Random** = K uniformly-random pool slugs (seeded, reproducible).
- **Metric** = mean recall@K = |selected ∩ useful| / |useful|, over 49 sessions.

## Result (seed 1)

| K | ranked recall | random recall | lift |
|---|---|---|---|
| 3 | 0.153 | 0.018 | ~8.5× |
| 5 | 0.240 | 0.018 | ~13× |
| 8 | 0.458 | 0.022 | ~20× |
| 12 | 0.637 | 0.045 | ~14× |

## Reading

- **The null hypothesis (ranking ≈ random) is decisively rejected** at every budget —
  ranked recall is 8–20× the random baseline.
- This holds on **static signals alone** (ref_count / effectiveness / recency), with the
  keyword-overlap component disabled. The live ranker, which adds prompt-keyword overlap,
  can only do better.
- **Cost framing:** at a 12-slot budget the ranker surfaces ~64% of a session's useful
  memories; dump-all would need the entire corpus for 100%. Ranking captures most of the
  value in a small, warm, cheap injection — the v2 thesis.
- The ranking/retrieval half of hypomnema is **validated, not commoditized** — native
  memory injects only an index and does no ranking; this lift is the differentiator.

## Caveats (honest scope)

- **Small n** (49 sessions; WAL compaction collapsed older injects into `inject-agg`,
  losing per-session inject linkage). Directional, not a precision verdict.
- **Static-only.** Keyword overlap (the live ranker's strongest signal) is untestable
  historically; its lift is unmeasured here.
- **First-time-useful slugs are missed by construction** — a slug with no pre-session
  signal ranks on zero-safe priors + slug tie-break (see `TestReplay_ColdStart…`). Static
  signals genuinely cannot predict a novel-but-useful fact; that's the keyword component's job.

## Use

Feeds scoring-weight calibration (the `rank` weights are fixed constants pending this kind
of evidence) and supports the spec's §1.1 decision to KEEP ranked injection while merging
the two pipelines. Re-run `memoryctl ab` as the corpus grows for a larger-n read.
