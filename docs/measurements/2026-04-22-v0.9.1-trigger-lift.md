# v0.9.1 trigger lift — replay measurement, 2026-04-22

## Question

Continuity from 2026-04-22 recorded a seeds-only replay baseline of **trigger-match 2.1%, shadow-miss 89.6%** on the 48-prompt synthetic corpus. The hypothesis (reviewer's, earlier): the substring-trigger channel is effectively dead when seed `triggers:` arrays are narrow or absent, while FTS5 body-match (shadow pass) surfaces the same records reliably — so populating triggers with natural-phrasing variants should raise primary retrieval.

This note records the measurement that tested the hypothesis.

## Method

1. Baseline: `scripts/replay-runner.sh` against a fresh fixture seeded from `seeds/` (7 files with narrow `triggers:` arrays — "shell escape", "path spaces", "git secret leak", …).
2. Corpus: `scripts/synthetic-corpus.txt`, 48 prompts across 7 domains.
3. Manual match: for each seed, identified the prompt(s) it semantically addresses (6 of 7 seeds had ≥1 natural-fit prompt; `source-url-rot-no-local-copy` had none).
4. Expansion: extended `triggers:` in each seed with phrasings drawn from the actual prompt text, not from the seed's own body (no overfitting to internal vocabulary).
5. Re-ran replay against the expanded fixture.
6. Cross-checked: verified `shadow-miss` for the matched slug correctly drops to 0 in the triggered prompts (hook contract regression guard).

## Results

| Metric | Baseline | Expanded | Δ |
|---|---|---|---|
| `trigger-match` prompts | 1/48 (2.1%) | **5/48 (10.4%)** | **+4 prompts, 5×** |
| `shadow-miss` prompts | 43/48 (89.6%) | 43/48 (89.6%) | 0 |
| Total `shadow-miss` events | 160 | 160 | 0 |
| Natural-fit ceiling | — | 6/48 (12.5%) | blocked by #33 |

### Why #33 still misses

The `" no "` substring inside `"no effect"` falls inside the ±40-char negation window (`hooks/user-prompt-submit.sh:178`). The heuristic treats the entire prompt as "the user is asking about something that is NOT the case", so all triggers are dropped. This is a genuine false-positive of the negation filter — the user's phrasing happens to collide with negation vocabulary that describes the problem, not the query. Out of scope for trigger tuning; open question for the negation heuristic.

### Shadow-miss suppression — contract check

For every prompt that triggered a slug in the expanded run:

```
#14 → triggered file-path-spaces-unquoted        → shadow-miss(file-path-spaces-unquoted) = 0
#17 → triggered shell-escape-special-chars       → shadow-miss(shell-escape-special-chars) = 0
#19 → triggered encoding-utf8-vs-cp1251          → shadow-miss(encoding-utf8-vs-cp1251) = 0
#20 → triggered datetime-naive-timezone          → shadow-miss(datetime-naive-timezone) = 0
#27 → triggered git-committed-secrets-history    → shadow-miss(git-committed-secrets-history) = 0
```

Per-slug shadow-miss correctly collapses when the primary pipeline injected that slug. The overall `shadow-miss` event count stayed at 160 because FTS5 BM25 still matches bodies of *other* seeds for each prompt (3-4 unrelated slugs per prompt on average) — that is a separate noise channel, not a regression.

## Interpretation

- **Primary retrieval is sensitive to trigger phrasing in the expected way.** Going from 2 narrow triggers to 6-8 phrasing-aware triggers lifted coverage by 5× for a modest edit (~30 added trigger strings across 6 files, no body changes).
- **The shadow pass is a working recall signal.** Every expanded match corresponded to a slug that shadow had already been flagging as a miss — the diagnostic was pointing at exactly the files that needed wider triggers.
- **Ceiling is bounded by the corpus, not the pipeline.** The remaining gap from 10.4% to 12.5% (1 prompt, #33) sits in negation heuristics, and 37 of 48 prompts have no natural-fit seed — expanding triggers further without adding seeds would overfit.

## Follow-ups

- Negation false-positive on problem-description `"no"` (#33). Separate investigation: either narrow the ±40 window, require a negation token adjacent to the trigger itself, or add a "problem-describing" allowlist for words like "no effect", "not working", "never fires".
- FTS body-match noise: ~3 unrelated slugs per prompt consistently surface. Worth a future filter, e.g. minimum BM25 score threshold or topic-similarity gate. Not blocking — shadow-miss is observational, does not enter context.
- Seed coverage for the 37 "unmatched" prompts is a corpus-vs-seed product question, not a retrieval-tuning one.
