# Audit 2026-04-16 — hypomnema v0.8

## Context

Hypomnema v0.8 shipped recently with heavy hardening (WAL locking, `pipefail`, UTF-8 via perl, regex escaping, portable stat, externalized constants, install wizard). The v0.9 plan to further decompose `session-start.sh` was deferred. This audit verifies that hardening is complete and surfaces residual risks before the next feature cycle.

## Scope

**In:**
- `hooks/` — all shell hooks (14 files)
- `hooks/lib/` — shared libraries (7 files)
- `bin/` — utility scripts (4 files)
- `templates/` — memory templates
- `docs/` — README, CLAUDE.md, QUICKSTART, TROUBLESHOOTING, FAQ, MIGRATION
- `install.sh` — installer

**Out:**
- `seeds/` (test data)
- `journal/` (auto-rotating logs)
- `~/.claude/memory/` content (user data, not code)
- `.git/`, `skills/` (third-party / out-of-project)
- `docs/plans/`, `docs/specs/` (historical design docs, not live code)

## Round budget

**3 rounds max.** Stop early if `rejected > fixed` in any round.

## Categories + acceptance criteria

### 1. Correctness
**What it covers:** Frontmatter parsing, scoring math (priority key v0.6, TF-IDF, WAL spaced repetition), lifecycle rotation (active/archived transitions), dedup logic.

**Acceptance criteria:**
- No div-by-zero, `log(0)`, off-by-one in scoring
- `sort -k` invocations produce deterministic ordering (stable on ties)
- Frontmatter parser handles the documented schema fields 1:1
- Injection ranking matches the order documented in CLAUDE.md (`project → status → severity → recency → log₁₀(ref_count) → recurrence → ref_count`)

### 2. Robustness
**What it covers:** Edge cases in memory files.

**Acceptance criteria:**
- Empty files, missing frontmatter, malformed YAML do not crash hooks (graceful skip + journal entry)
- CRLF, BOM, trailing whitespace in frontmatter values are tolerated
- Unicode in keywords/description (Cyrillic, emoji) preserved through scoring and injection
- Empty arrays (`keywords: []`, `domains: []`) do not break parsing
- Multi-line frontmatter values (folded/block scalars) either work or fail cleanly

### 3. Security
**What it covers:** Injection vectors from user-controlled memory content into shell/awk/perl/jq.

**Acceptance criteria:**
- No unquoted variable expansion where value originates from memory file content
- No `eval` / `source` on user-controlled data
- Slug/filename used as path argument is validated (no `..`, no absolute paths, no NUL)
- Keywords used as regex are escaped before passing to `grep` / `perl` / `awk`
- Description / body text injected into final context is escaped against prompt injection markers (e.g. `</system-reminder>`)

### 4. Performance
**What it covers:** Startup latency on large memory dirs.

**Acceptance criteria:**
- `session-start.sh` total wall time on a synthetic 500-file memory ≤ 1.5s on developer machine (baseline to be measured, not regressed)
- No O(n²) scans where O(n) works
- `bench-memory.sh` runs clean and produces a reproducible baseline

### 5. Documentation drift
**What it covers:** Consistency between docs and code.

**Acceptance criteria:**
- README, CLAUDE.md (project + global), QUICKSTART reference the actual priority key formula, actual caps, actual directory names as implemented in `hooks/lib/score-records.sh` and `hooks/lib/build-context.sh`
- Versioned claims in docs match the current `CHANGELOG.md` entry
- Frontmatter field list in docs matches fields actually parsed by `hooks/lib/parse-memory.sh`
- Examples in README compile/parse with current hooks

## Exit condition

Audit is **done** when:
1. All three rounds complete, OR a round has `rejected > fixed` and we stop early
2. Every `status: needs_review` finding has a decision (fix / wont-fix / follow-up)
3. Full test suite (`hooks/test-memory-hooks.sh`) passes on main
4. A single PR is opened with categorized commits and rollback script

## Follow-up: runtime audit (Phase 7)

After the code audit lands, a separate runtime audit verifies that live memory (`~/.claude/memory/`) behaves as designed:
- Every file has a parseable frontmatter with required fields
- WAL grows on injection, compacts on rotation, no orphans
- Dedup actually removes near-duplicates (no silent duplicates by keyword set)
- Outcome tracking flips `success_count` and `recurrence` after positive/negative signals
- Injection on a real session for this project produces the expected set (priority key order observable in the rendered context)

This is separate from code-level correctness — even correct code can produce stale or malformed memory if hooks were skipped in past sessions.

## Anti-scope (do NOT do in this audit)

- Do not refactor `session-start.sh` decomposition (that is v0.9 deferred work)
- Do not add new features
- Do not rewrite doc sections unless they are factually wrong
- Do not rename frontmatter fields (breaks existing user memories)
- Do not change the priority key formula (requires separate design review)
